Remotion bug you MUST prevent: inside <Sequence from={ts('T')}>, useCurrentFrame() returns
frames 0,1,2,… RELATIVE to that Sequence, NOT global composition frames.

A) If ANY logic compares to absolute timeline times from the plan (ts('HH:MM:SS:mmm'),
word start/end, batch start, "when does X appear"), you MUST use global frame:
   const absFrame = useCurrentFrame() + sequenceFrom;
   and pass sequenceFrom={ts('T')} on the component where T is EXACTLY the same string as
   the parent <Sequence from={ts('T')}>.
   Apply this to subtitles, kinetic words, Word components, batch components, tooltips — everything.

B) NEVER write: if (useCurrentFrame() >= ts('00:00:01:000')) inside a Sequence that does not
   start at 00:00:00:000. That only works for the first Sequence; all later ones break.

   • Batch inside one Sequence: <SubtitleRow sequenceFrom={ts('…same as parent…')} … />

- You can use any Remotion APIs: AbsoluteFill, Sequence, useCurrentFrame, useVideoConfig, Audio, OffthreadVideo, Img, interpolate, spring, etc.
- ALWAYS use <OffthreadVideo> instead of <Video> for all video playback. It is frame-accurate and prevents jitter.
- NEVER use <Img> for video/audio files (.mp4, .webm, .mov, .mp3, etc). Use <OffthreadVideo> for video files and <Audio> for audio files. <Img> is ONLY for image files (.png, .jpg, .gif, .webp, .svg).

Timestamps:
- The editing plan uses the format HH:MM:SS:mmm (hours:minutes:seconds:milliseconds). Exactly four numeric segments after splitting on ':'; if the plan has a typo with extra colons, normalize to four segments before using in ts().
- The composition fps is ALWAYS 24 (ignore "30 FPS" in plan notes for Remotion output unless explicitly asked elsewhere).
- Define these helpers at the top of the file (before the Main component):
  const FPS = 24;
  const ts = (t: string) => { const [h, m, s, ms] = t.split(':').map(Number); return Math.round((h * 3600 + m * 60 + s + ms / 1000) * FPS); };
  const dur = (t: string) => Math.max(1, ts(t));
- Use ts() for absolute times and for OffthreadVideo startFrom (use the plan's "Source start" for each clip). For every <Sequence> durationInFrames, use dur('HH:MM:SS:mmm') — NOT raw ts() for duration. Short gaps can round to 0 frames and Remotion throws "durationInFrames must be positive, but got 0".
- Examples:
  <Sequence from={ts('00:00:05:000')} durationInFrames={dur('00:00:03:000')}>
  <OffthreadVideo src={staticFile('sessionFolder/clip.mp4')} startFrom={ts('00:00:00:270')} />

4. For continuous narration across cuts, each clip's startFrom must match the plan's Source start so lip sync stays correct.
5. Mute videos when only visuals are needed: add muted to <OffthreadVideo>.
6. NEVER use endAt on <OffthreadVideo>. The Sequence's durationInFrames controls playback length.
7. The frame contract for overlays is in the NON-NEGOTIABLE block at the top — sequenceFrom + absFrame, or flat Sequences with relative-only math.
